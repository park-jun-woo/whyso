# 수정지시서 001: whyso history --grep 옵션 추가

## 목적

`whyso history <file> --grep <keyword>` — 히스토리 엔트리 중 keyword가 포함된 것만 필터링 출력.

## 배경

Claude Code hook(PreToolUse/PostToolUse)에서 Read 시점에 자동 호출하려는 용도.
hook stdin으로 `transcript_path`가 들어오므로 최근 Grep 키워드를 추출할 수 있고,
`whyso history <file> --grep <keyword>`로 관련 히스토리만 리턴하면 노이즈 없이 컨텍스트 주입 가능.

## 동작

1. `--grep` 플래그가 없으면 기존 동작 그대로 (전체 히스토리 출력)
2. `--grep <keyword>` 있으면 각 히스토리 엔트리의 `user_request`, `answer` 필드에서 keyword를 case-insensitive 검색
3. 매칭되는 엔트리만 출력
4. 매칭 없으면 빈 출력 (exit 0)

## 사용 예시

```bash
# 수동 사용
whyso history internal/gluegen/hurl.go --grep substitutePathParams

# Claude Code hook에서 사용
# Read hook이 transcript에서 최근 Grep 키워드 추출 후:
whyso history "$file" --grep "$keyword" 2>/dev/null
```

## 변경 범위

- CLI 플래그 파싱에 `--grep` 추가
- history 출력 로직에 필터링 조건 추가
- 캐시 yaml 읽기 로직은 변경 없음

## 테스트

1. `--grep` 없이 기존 동작 유지 확인
2. `--grep`로 매칭되는 엔트리만 출력 확인
3. 매칭 없을 때 빈 출력 + exit 0 확인
4. case-insensitive 검색 확인
