# 수정지시서 002 실행결과: internal → pkg 공개 API 추출

## 상태: 완료

## 실행 내용

### 1. pkg/ 디렉토리 생성 및 패키지 이동

| 이동 전 | 이동 후 |
|---|---|
| `internal/codemap/` | `pkg/codemap/` |
| `internal/history/` | `pkg/history/` |
| `internal/model/` | `pkg/model/` |
| `internal/parser/` | `pkg/parser/` |
| `internal/output/` | 이동 안 함 (CLI 전용) |

### 2. import 경로 변경

| 파일 | 변경 내용 |
|---|---|
| `pkg/history/builder.go` | `internal/parser` → `pkg/parser` |
| `pkg/history/chain.go` | `internal/model` → `pkg/model` |
| `pkg/parser/extract.go` | `internal/model` → `pkg/model` |
| `pkg/parser/jsonl.go` | `internal/model` → `pkg/model` |
| `internal/output/formatter.go` | `internal/history` → `pkg/history` |
| `internal/output/reader.go` | `internal/history` → `pkg/history` |
| `cmd/whyso/main.go` | `internal/{codemap,history,parser}` → `pkg/{codemap,history,parser}` |

### 3. 구 internal 패키지 삭제

`internal/codemap`, `internal/history`, `internal/model`, `internal/parser` 삭제. `internal/output`만 유지.

## 검증 결과

| 검증 항목 | 결과 |
|---|---|
| `go build ./cmd/whyso/...` | 성공 |
| `whyso sessions` | 정상 출력 |
| `whyso map` | 정상 출력 (`# whyso/v1` 헤더 포함) |
| `whyso history <file>` | 정상 출력 (`apiVersion: whyso/v1` 포함) |

## 최종 디렉토리 구조

```
whyso/
├── cmd/whyso/main.go          # CLI — pkg/ + internal/output import
├── pkg/
│   ├── codemap/               # 공개: keyword map
│   ├── history/               # 공개: 파일 이력
│   ├── model/                 # 공개: 데이터 타입
│   └── parser/                # 공개: JSONL 파싱
├── internal/
│   └── output/                # 비공개: CLI 전용 YAML/JSON 포매터
└── go.mod
```
